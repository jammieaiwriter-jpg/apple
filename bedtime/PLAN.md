# 定案與建造計畫

> 來源：2026-06-14 多 agent 討論群（提案→內容→實施→取捨 四階段收斂）。
> ⚠️ 須與 [`../LEARNING_DNA.md`](../LEARNING_DNA.md) 對齊（自尊保護、善用記憶優勢、動態適應）。

## ✅ 已達成共識
1. **技術架構**：純靜態 HTML + GitHub Pages（apple repo 子路徑 `/bedtime/`），零成本。
2. **資料層**：故事內容存 JSON；每句直接保存 `text` 與實際 `zhuyin`，不使用沒有內容來源的注音旗標。
3. **UI**：聊天泡泡逐句顯示，一次一句，降低六歲閱讀壓力。
4. **朗讀**：`speechSynthesis`，`lang: 'zh-TW'`、`rate: 0.9`（可在手機調）。
5. **互動**：每幕一個目標詞；幕末二選一按鈕；`localStorage` 存 `{ week, act, choice: "A"/"B" }`；隔天旁白對應 `next_hint_A/B`。
6. **首版不做**：錄音、詞彙動畫、答題式熟練度追蹤。只保存詞彙與故事聯想，供自然重現與週末重述。
7. **內容來源**：AI 預先生成故事 JSON，家長只選主題。
8. **建造順序**：先完成第一幕的泡泡、朗讀、秘密情報與 choice 完整互動 → **立即手機驗收**注音字型、語速與按鈕理解 → 再驗證第二幕回應旁白 → 補滿五幕與週末重述。

## 內容審閱規則
- 採 `pre_generated_verified_only`：每週 JSON 上線前必須由成人檢查故事價值觀、詞彙難度、注音與無對錯式語氣。
- JSON 必須標記 `"review_status": "adult_verified"`；未標記的故事不列入正式故事庫。

## 🍎 LEARNING_DNA 對齊（討論時未納入，務必補上）
- **自尊保護**：二選一一律讓 Apple 替主角決定，**禁止**任何對錯、分數與過關字樣；三層提示改稱「秘密情報」。
- **記憶優勢**：每幕保存 `target_word` 與 `word_connection`，選擇後加入本機詞彙寶盒；隔天旁白與週末故事地圖自然重現。
- **動態適應**：首頁自動跳到下一個未完成幕，已完成內容不強迫重播；Apple 仍可從進度列主動回看喜歡的幕。

## 故事 JSON 結構（每幕一個 act 物件）
```json
{
  "id": "week01-act01",
  "act": 1,
  "act_title": "河邊的新朋友",
  "lines": [{ "text": "小兔走到河邊。", "zhuyin": "ㄒㄧㄠˇ ㄊㄨˋ ㄗㄡˇ ㄉㄠˋ ㄏㄜˊ ㄅㄧㄢ。" }],
  "target_word": "勇敢",
  "word_connection": "勇敢不是完全不怕，而是願意慢慢往前。",
  "prompt_question": "你覺得小熊應該…",
  "choice_A": "自己去找",
  "choice_B": "等媽媽回來",
  "next_hint_A": "昨天你說小熊要自己去找，看他遇到什麼！",
  "next_hint_B": "昨天你說要等媽媽，媽媽真的回來了嗎？",
  "secret_hints": ["觀察：…", "方法：…", "關鍵：…"]
}
```
一個 `weekNN.json` = 5 幕，每幕 ≤ 60 字。

## 建造步驟（逐步 /do，邊做邊驗收）
1. **今天**：產 `stories/week01.json`（主題「友情」，5 幕）＋讀取器 `index.html`；先完成第 1 幕完整互動：泡泡逐句＋朗讀＋秘密情報＋A/B 按鈕＋存 `localStorage`。
2. **手機驗收**：iOS Safari 上測注音字型大小與 `rate`，不對立即調，再往下。
3. **第 2 幕**：讀取前一幕 choice，插入 `next_hint_A/B` 旁白——驗證「故事記得她昨天說的話」。
4. **補滿幕 3–5**。
5. **週末重述頁**：五幕標題縮圖＋家長勾「提示 / 獨立」，純靜態。

家長操作上限：每週在下拉選單選一個主題，其餘全自動。
