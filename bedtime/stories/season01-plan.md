# 第一季睡前故事計畫

## 內容節奏：主題 → 切點 → 故事

- 一週一個核心詞主題；每個主題固定拆成 **5 個「切點」(facet)**，每個切點是一個明確的教學角度，由角色行動演出（不說教）。
- **一切點 = 一篇故事**（1:1）。切點有順序，組成一個溫和的進程；夜間頁在本週主題內每晚自動換一篇 `adult_verified` 故事，照切點順序輪播，但每篇仍可單獨聽懂。
- 孩子一週聽五晚，5 個切點依序形成「開始理解 → 遇到困難 → 顧及他人或界線 → 更成熟的行動 → 帶進生活」。
- 命名：`weekNN`（切點一／主打）、`weekNN-2`、`weekNN-3`…。新增故事預設 `pending_adult_review`，成人試聽通過才進夜間輪播。
- 資料：`catalog.json` 的每週 episode 用 `facets` 宣告切點清單，每篇故事用 `focus` 標記它對應哪個切點。
- 抽象概念必須轉成六歲孩子看得見的具體行動；安全與可信的角色行為優先於硬套五晚節奏。
- 不強迫眼神接觸、立即原諒、和好或留在傷害者附近；孩子感到害怕或不安全時，可以直接離開並找可信任的大人。

## 故事配方表（寫稿前先抽配方）★ 核心流程

每篇故事**先抽一張配方卡、再照配方寫**——讓變化來自設計而非運氣。工具：`tools/draw-recipe.py`（零 token）。

```
python3 tools/draw-recipe.py --facet "<本篇切點>" --theme <核心詞> --id <weekNN-x>
```

配方卡欄位（多軸交叉輪替）：

| 欄位（JSON 鍵） | 說明 | 是否硬閘 |
|---|---|---|
| `focus` | 切點，由各週大綱固定 | ✅ 必屬本週 facets |
| `shape` | 故事形狀（8 種） | ✅ 不撞前一晚 |
| `protagonist` | 主角（**2026-06 起全放開，不再鎖森林**：孩子／動物／玩具／自然物／食物／小精靈／交通工具…） | ✅ 必填 |
| `scene` | 場景（家中／自然／微小世界／天空／季節／生活場所／夢境…） | ✅ 必填 |
| `emotion_arc` | 情緒曲線（起→轉→**收回平靜**；睡前結尾一律安心，驚喜只能溫的） | ✅ 必填 |
| 核心事件 | 今晚發生什麼小事（依切點，寫進 review_note/正文） | — |
| `resolution` | 解決方式（觀察／等待／合作／分享／修補／唱歌／擁抱／想像／放下／傾聽／深呼吸） | ✅ 不撞前一晚 |
| 陪伴角色 | 誰陪主角（常是不說話的物件；會說話才占聲線） | — |
| `dominant_sense` | 主導感官（視／聽／觸／嗅／味） | ✅ 不撞前一晚 |
| 睡前收束 | 這篇怎麼慢慢靜下來 | — |
| `ending_style` | 結尾句式（願式／比喻式／物件呼應式／身體放鬆式／明日式） | ✅ 不撞前一晚 |
| 禁用元素 | **自動**由最近兩晚帶入（不要同形狀動作／同場景／同解法收尾） | 由 draw-recipe 產生 |

八種形狀：旅程／相遇／等待／製作／尋找／來訪／**變化**（角色慢慢轉變）／**守護**（照顧更小更弱的對象）。
四個「不撞前一晚（含跨週）」的硬閘軸 = `shape × resolution × dominant_sense × ending_style`，由 `check-bedtime-story.py` 強制。

### 五種結尾句式（輪替，破除晚安套路）
- 願式：「雨芯，晚安。願～，做一個～的夢。」
- 比喻式：「雨芯，晚安。今晚就像～，～。」
- 物件呼應式：「雨芯，晚安。～也～了，妳也～。」
- 身體放鬆式：「雨芯，晚安。讓～慢慢～，～地睡。」
- 明日式：「雨芯，晚安。～還在那裡等明天，先安心睡吧。」

## 架構矩陣（故事形狀）

- 問題：早期友情兩篇〈星光小船〉〈荷葉小傘〉六段逐拍同骨架（夜晚到場 → 物件起程出狀況 → 走到暗處猶豫 → 幫一隻路過小生物 → 雨停月出 → 走回家收物件晚安），第二晚就膩。
- 解法：每篇先選一個**故事形狀（shape）**，夜間輪播保證**相鄰兩晚不同形**、**主導感官輪替**；六段與「平靜收尾」不變，變的是中間怎麼走。

| 形狀 | 走法 | 適合的切點類型 |
|---|---|---|
| 旅程型 | 出發 → 難關 → 幫助 → 回家 | 初次靠近、嘗試、選路 |
| 相遇型 | 兩角色各做各的 → 小誤會 → 對話化解 → 一起完成一件事 | 分享、友善、道歉、表達 |
| 等待型 | 安靜守候一件事發生，低動作、重感官 | 耐心、傾聽、陪伴 |
| 製作型 | 一起動手做／修 → 過程出錯 → 調整完成 | 合作、嘗試 |
| 尋找型 | 循線索找遺失的東西或聲音來源 | 傾聽、選擇、想像 |
| 來訪型 | 固定一個場景不移動，純靠對話推進 | 表達、友善、誠實 |

規則：同一週五晚的形狀不重複；下一週可重用，但不與「前一晚」同形。對白為主的篇章（相遇／來訪型）優先採多聲線。

## 點題前言（prologue）

- 每篇 `intro` 之後、第一段之前，加 4–6 句 `prologue`，作為六歲孩子的**故事入口**：先簡介今晚會出現的角色、故事開始的場景、角色一開始遇到的小狀況，再溫柔明講今晚的核心道理。
- `prologue` 不要像課堂摘要，也不要劇透全部解法；孩子聽完應知道「誰在什麼地方，要開始一件什麼事」。
- 結論的核心道理放在前言；故事本體仍以角色行動「演出」，不在正文裡反覆說教。

## 尾端收束

- 第 6 段不能只快速說「大家懂了、晚安」。需先落下角色的具體改變，再給一個安靜畫面，讓故事自然滑進 `wind_down`。
- `wind_down.scene` 可簡短回望故事結果：角色已經怎麼安定下來、場景或物件留下什麼溫柔變化；接著 `breath` 放慢身體，`goodnight` 才正式道晚安。
- 收束是睡前畫面，不是主題複習；避免格言式結尾。

## 多聲線（voices / turns）

- 音檔採「旁白 + 兩個童聲」。故事檔加 `voices`（role → Azure 語音）與 `sections[].turns`（`[{role,text}]`）。
- **定案聲線（2026-06-16，Emma 試聽通過）**：
  - `narrator` → `zh-TW-HsiaoChenNeural`（台灣女聲，沉穩旁白）
  - `mimi` → `zh-CN-XiaoshuangNeural`（女童・主角女）
  - `dodo` → `zh-CN-YunxiaNeural`（男童）
  - `girl2` → `zh-CN-XiaoyiNeural`（第二女童，2026-06-16 Emma 試聽選定）— 只在「同篇出現兩個說話的女生」時使用，用來和 `mimi` 區分。
  - zh-TW 無童聲，真童聲只在 zh-CN，故角色帶輕微大陸口音；Emma 已接受。旁白維持台灣腔。prosody 維持 `-15%`。
- 角色↔role 對應：主角女→`mimi`、第二個說話的女生→`girl2`、男角→`dodo`。一篇最多這四個聲線（睡前低刺激優先）；同性別的次要配角若不與主角同場對話，可共用同一聲線。
- 對白為主：角色說自己的話，**旁白只接場景與轉場**，不替角色轉述心情。
- `sections[].text` 仍保留整段純敘述（= turns 串接，不含 role 標記），給瀏覽器後備與閱讀；現有舊篇無 `turns` 時沿用單聲。

### 寫稿前企劃欄位

每篇開寫前先確認：`facet`、`shape`（故事形狀）、`visible_action`、`body_signal`、`boundary_or_safety`、`dominant_sense`、`change_action`、`prologue`（角色＋場景＋起始狀況＋核心道理）、第 6 段收束畫面、`voices`／`turns`（多聲線分工）。

## 共通規格

- 共用森林世界，但每集獨立可懂；主角依衝突選擇，不固定輪替。
- 維持 `schema_version: 2`、六個 `sections` 與 `wind_down`。
- 核心詞由角色行動呈現；週末問題問角色做了什麼，不考名詞定義。
- 故事文字遵循 `docs/tts-audio-writing-guide.md`：用短句、自然停頓、適量口語助詞、疊字與聲音詞營造真人說故事的呼吸感。
- 老山羊長故事嵌入既有段落，不新增第七段；成語僅為可選延伸。
