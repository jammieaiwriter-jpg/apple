# TTS 聽覺優化寫作準則

本專案的故事會優先交給 Azure Speech `zh-TW-HsiaoChenNeural` 朗讀，Azure 無法使用時才退回瀏覽器 `speechSynthesis`。技術定案與 Claude 後續套用規則請見 [Azure Neural 睡前故事朗讀定案](azure-neural-narration-handoff.md)。文字不只要適合閱讀，也必須讓六歲孩子閉上眼睛聽時，感覺像有人溫柔地在床邊說故事。

## 核心目標

- 聲音溫柔、放鬆、低刺激，越接近結尾越安靜。
- 語句具有真人說故事的呼吸感，不像課本或說明書。
- 保留台灣日常口語，但不刻意堆疊語助詞。
- 聲音設計服務情節與情緒，不用符號或驚嘆製造突然的高刺激。

## 標點與停頓

- 交替使用短句、逗號與刪節號「……」，讓重要畫面與情緒轉折有呼吸空間。
- 減少連續、等長、只用句號結尾的句型。
- 不在故事正文寫出「（空格）」；TTS 可能把文字直接念出來。
- 波浪號 `~` 只可少量使用，且必須經實際試聽確認不會被念出或造成怪異停頓。
- 長留白仍由播放程式控制：段落後 1.3 秒、收心前 1.9 秒、收心段間 2.3 秒。

## 口語與句型

- 使用六歲孩子聽得懂的台灣口語，避免生硬書面語、抽象詞與不必要的成語。
- 可自然加入「呢、喔、呀、耶、囉、嘛、啊」，但不能每句都加，也不能改變角色原本個性。
- 多使用適量疊字，如「小小的、靜靜的、輕輕的、慢慢的」，幫助降低朗讀速度。
- 句子以一次能舒服聽懂一個畫面或動作為原則；過長句應拆開。

## 聲音與氣音

- 每篇故事至少自然加入 2–3 個能帶動情緒的聲音詞或氣音，例如「呼……」「咦？」「噓……」「鈴、鈴」「沙沙」。
- 聲音詞必須符合場景，不能為達數量硬塞。
- 睡前故事避免高亢、突然或連續的驚嘆，例如大量使用「哇！」或尖銳碰撞聲。
- 同篇故事應有一組主要聲音意象，並與同週其他故事區隔。

## 試聽檢查

文字評分只能預檢，不能取代公開網頁使用 `zh-TW-HsiaoChenNeural` 的實際試聽。成人試聽時確認：

1. 標點是否產生自然停頓，而非奇怪斷句。
2. `……`、`~` 與聲音詞是否被正確朗讀。
3. 語助詞是否自然，沒有刻意裝可愛或過量。
4. 中後段是否逐漸變慢、變輕，能自然進入 `wind_down`。
5. 有沒有書面腔、難念句或讓六歲孩子聽不懂的用詞。

## 範例

較生硬：

> 米米看見月亮出來了。她拿著風鈴坐在旁邊等待。

較適合 TTS：

> 米米輕輕抬起頭……咦？月亮從樹梢後面，慢慢露出來了呢。她抱著小風鈴，靜靜坐在草地旁，聽著鈴、鈴的聲音……再等一下下。
