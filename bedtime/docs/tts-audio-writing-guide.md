# TTS 聽覺優化寫作準則

本專案的故事會優先交給 Azure Speech `zh-TW-HsiaoChenNeural` 朗讀，Azure 無法使用時才退回瀏覽器 `speechSynthesis`。技術定案與 Claude 後續套用規則請見 [Azure Neural 睡前故事朗讀定案](azure-neural-narration-handoff.md)。文字不只要適合閱讀，也必須讓六歲孩子閉上眼睛聽時，感覺像有人溫柔地在床邊說故事。

## 核心目標

- 聲音溫柔、放鬆、低刺激，越接近結尾越安靜。
- 語句具有真人說故事的呼吸感，不像課本或說明書。
- 故事不只哄睡，也要讓孩子吸收優美的中文寫作語感；五感描寫以比喻、擬人、具體動作與光影質地為主。
- 保留台灣日常口語，但不刻意堆疊語助詞。
- 聲音設計服務情節與情緒，不用符號或驚嘆製造突然的高刺激。

## 標點與停頓

- 交替使用短句、逗號與刪節號「……」，讓重要畫面與情緒轉折有呼吸空間。
- 減少連續、等長、只用句號結尾的句型。
- 不在故事正文寫出「（空格）」；TTS 可能把文字直接念出來。
- 波浪號 `~` 只可少量使用，且必須經實際試聽確認不會被念出或造成怪異停頓。
- 長留白仍由播放程式控制：段落後 0.9 秒、收心前 1.9 秒、收心段間 2.3 秒。

## 口語與句型

- 使用六歲孩子聽得懂的台灣口語，避免生硬書面語、抽象詞與不必要的成語。
- 可自然加入「呢、喔、呀、耶、囉、嘛、啊」，但不能每句都加，也不能改變角色原本個性。
- 多使用適量疊字，如「小小的、靜靜的、輕輕的、慢慢的」，幫助降低朗讀速度。
- 句子以一次能舒服聽懂一個畫面或動作為原則；過長句應拆開。

## 故事入口與收束

- `prologue` 不只說道理，也要替六歲孩子開一扇故事門：先簡介今晚會出現的角色、開始的場景，以及角色一開始遇到的小狀況，再用一兩句溫柔帶出核心道理。
- `prologue` 建議 4–6 句，約 120–220 字；不要像課堂摘要，也不要劇透全部解法。孩子聽完應該知道「誰在什麼地方，要開始一件什麼事」。
- `prologue` 的順序建議為：角色與場景 → 起始狀況 → 期待孩子聽見的情緒或選擇 → 核心道理。
- 故事正文最後一段不能潦草結束。第 6 段需先讓角色的變化落地，再給一個安靜畫面，最後自然滑進 `wind_down`。
- `wind_down.scene` 可以簡短回望故事結果：角色已經怎麼安定下來、物件或場景留下什麼溫柔變化，再進入呼吸與晚安。這是睡前收束，不是主題複習。

## 聲音與氣音

- 狀聲詞只作點綴，不作五感描寫的主體。每篇保留一組主要聲音意象即可，通常 3–5 次足夠；聽覺主導故事也不應逐段堆效果音。
- 聲音詞必須符合場景，不能為達數量硬塞；不要讓「沙沙、刷刷、嗒嗒、喀」一路排開，像在朗讀效果音清單。
- 同一個聲音詞若反覆出現，必須有情緒變化：例如從硬硬的敲擊，變成輕輕的提示；不能只是重複標記動作。
- 睡前故事避免高亢、突然或連續的驚嘆，例如大量使用「哇！」或尖銳碰撞聲。
- 同篇故事應有一組主要聲音意象，並與同週其他故事區隔。

## 文學化五感

- 五感描寫優先使用**比喻法**：把聲音、觸感、氣味、光線寫成孩子能想像的畫面，例如「風從小沙堆旁邊走過，像把一張薄薄的毯子掀起一角」。
- 優先使用**擬人法**讓物件溫柔地動起來，例如「小河把星光輕輕抱著」、「路燈把兩個影子慢慢放長」。擬人要安靜、自然，不要變成誇張卡通。
- 優先用具體動作和身體感受承接情緒，例如「車身的熱一點一點退下去」、「紙肩慢慢平了」，少用直接說明「他明白了」「他覺得很開心」。
- 每段至少有一個清楚畫面，但不必每段都放狀聲詞。視覺、觸覺、嗅覺和空氣感，常比效果音更適合睡前朗讀。
- 正文避免作文講義式的結論句；把道理留在 `prologue`，正文讓畫面自己說話。

## 試聽檢查

文字評分只能預檢，不能取代公開網頁使用 `zh-TW-HsiaoChenNeural` 的實際試聽。成人試聽時確認：

1. 標點是否產生自然停頓，而非奇怪斷句。
2. `……`、`~` 與聲音詞是否被正確朗讀。
3. 語助詞是否自然，沒有刻意裝可愛或過量。
4. 中後段是否逐漸變慢、變輕，能自然進入 `wind_down`。
5. 有沒有書面腔、難念句或讓六歲孩子聽不懂的用詞。

## 範例

較生硬：

> 米米看見月亮出來了。她拿著風鈴坐在旁邊等待。

較適合 TTS：

> 米米輕輕抬起頭……咦？月亮從樹梢後面，慢慢露出來了呢。她抱著小風鈴，靜靜坐在草地旁，聽著鈴、鈴的聲音……再等一下下。

較生硬：

> 沙坑裡滿滿都是沙沙、刷刷、嗒嗒、喀喀的聲音。拍拍覺得聲音有點孤單。

較適合 TTS 與寫作語感：

> 剛才熱鬧的沙坑，慢慢空了下來。風從小沙堆旁邊走過，像把一張薄薄的毯子掀起一角。拍拍低頭看著自己的影子，忽然覺得，城堡還在，可是一起玩的暖意不見了。

較簡短的前言：

> 今晚的故事，是關於「用邀請代替命令」。想一起做一件事時，說出邀請，朋友的心比較容易靠近。

較適合六歲孩子的故事入口：

> 今晚的故事，會來到公園角落的一個小沙坑。沙坑裡有一支小指揮棒拍拍，還有三把想一起玩的小鏟子。拍拍很想快快蓋好城堡，所以一開始，他說話有一點像命令。等一下，我們會聽見小鏟子們的聲音怎麼慢慢變小，也會看見拍拍怎麼學會把「你要這樣做」換成「你想不想一起試試看」。一起玩時，邀請比命令更容易讓朋友留下來。

較潦草的尾端：

> 他們都知道邀請比較好。沙坑安靜了。晚安。

較完整的睡前收束：

> 小鏟子們靠在一起，沙坑裡的小城堡不高，卻有池塘、圓牆和貝殼門。拍拍沒有再急著安排下一步，只把最後一顆小石子放在門邊，等大家一起看。路燈把影子放得很柔，晚風也把沙面輕輕撫平。今晚的小城堡，像一個留著空位的邀請，安安靜靜等著明天。

## 對白與多聲線（2026-06 新模式）

故事改為「旁白 + 兩個童聲」的對白格式：`sections[].turns` 以 `role` 分配給 `narrator`／`mimi`／`dodo`，逐輪用不同 Azure 語音合成再串接（定案聲線見 `stories/season01-plan.md`）。寫作時：

- **讓角色自己說話**：心情與想法盡量用對白帶出，**旁白只接場景、動作與轉場**，不要替角色把話講完、也不要事後轉述「他覺得很開心」。
- **口吻分得開**：兩個角色講話的習慣要有區別（例如一個急、一個慢；一個愛問問題、一個愛形容），孩子閉著眼也能分出是誰在說。
- **一輪一個意思**：每個 `turn` 講一句到兩句、一個畫面或一個動作就換人，避免長段獨白；輪與輪之間程式會自動補約 0.25 秒停頓。
- **低刺激優先**：童聲本身較亮，睡前不要再寫高亢、搶白、連續驚嘆的對話；越靠近結尾，對白越少、旁白越輕，自然滑進 `wind_down`。
- **`text` 要等於 turns 串接**：每段的純文字 `text` 必須是該段所有 `turn` 的 `text` 依序相接（不含 role 標記），供瀏覽器後備與閱讀；用 `season01-plan.md` 約定的方式產生，避免兩者不同步。
- **童聲口音**：`mimi`／`dodo` 為 zh-CN 童聲、帶輕微大陸口音；避免寫只在台灣語境才自然、換成大陸腔會走味的字詞（必要時挑兩岸都自然的說法）。
