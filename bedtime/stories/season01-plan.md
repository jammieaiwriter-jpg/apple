# 第一季睡前故事計畫

## 內容節奏：主題 → 切點 → 故事

- 一週一個核心詞主題；每個主題固定拆成 **5 個「切點」(facet)**，每個切點是一個明確的教學角度，由角色行動演出（不說教）。
- **一切點 = 一篇故事**（1:1）。切點有順序，組成一個溫和的進程；夜間頁在本週主題內每晚自動換一篇 `adult_verified` 故事，照切點順序輪播，但每篇仍可單獨聽懂。
- 孩子一週聽五晚，5 個切點依序形成「開始理解 → 遇到困難 → 顧及他人或界線 → 更成熟的行動 → 帶進生活」。
- 命名：`weekNN`（切點一／主打）、`weekNN-2`、`weekNN-3`…。新增故事預設 `pending_adult_review`，成人試聽通過才進夜間輪播。
- 資料：`catalog.json` 的每週 episode 用 `facets` 宣告切點清單，每篇故事用 `focus` 標記它對應哪個切點。
- 抽象概念必須轉成六歲孩子看得見的具體行動；安全與可信的角色行為優先於硬套五晚節奏。
- 不強迫眼神接觸、立即原諒、和好或留在傷害者附近；孩子感到害怕或不安全時，可以直接離開並找可信任的大人。

## 架構矩陣（故事形狀）

- 問題：早期友情兩篇〈星光小船〉〈荷葉小傘〉六段逐拍同骨架（夜晚到場 → 物件起程出狀況 → 走到暗處猶豫 → 幫一隻路過小生物 → 雨停月出 → 走回家收物件晚安），第二晚就膩。
- 解法：每篇先選一個**故事形狀（shape）**，夜間輪播保證**相鄰兩晚不同形**、**主導感官輪替**；六段與「平靜收尾」不變，變的是中間怎麼走。

| 形狀 | 走法 | 適合的切點類型 |
|---|---|---|
| 旅程型 | 出發 → 難關 → 幫助 → 回家 | 初次靠近、嘗試、選路 |
| 相遇型 | 兩角色各做各的 → 小誤會 → 對話化解 → 一起完成一件事 | 分享、友善、道歉、表達 |
| 等待型 | 安靜守候一件事發生，低動作、重感官 | 耐心、傾聽、陪伴 |
| 製作型 | 一起動手做／修 → 過程出錯 → 調整完成 | 合作、嘗試 |
| 尋找型 | 循線索找遺失的東西或聲音來源 | 傾聽、選擇、想像 |
| 來訪型 | 固定一個場景不移動，純靠對話推進 | 表達、友善、誠實 |

規則：同一週五晚的形狀不重複；下一週可重用，但不與「前一晚」同形。對白為主的篇章（相遇／來訪型）優先採多聲線。

## 點題前言（prologue）

- 每篇 `intro` 之後、第一段之前，加 2–3 句 `prologue`：**明講今晚的小道理**，再開始故事（家長指定明講，非鏡頭式）。
- 結論放在前言；故事本體仍以角色行動「演出」，不在正文裡再說教。

## 多聲線（voices / turns）

- 音檔採「旁白 + 兩個柔和角色聲線」。故事檔加 `voices`（role → Azure 語音）與 `sections[].turns`（`[{role,text}]`）。
- 起步聲線：`narrator` HsiaoChen、`mimi` HsiaoYu、`dodo` YunJhe；prosody 維持 `-15%`，試聽若太亮再降。
- `sections[].text` 仍保留整段純敘述（= turns 串接），給瀏覽器後備與閱讀；現有舊篇無 `turns` 時沿用單聲。

### W02–W12 切點表

| 週次 | 切點 1 | 切點 2 | 切點 3 | 切點 4 | 切點 5 |
|---|---|---|---|---|---|
| W02 誠實 | 分清楚自己真正看見的事 | 犯錯後勇敢說出真相 | 把事情完整說清楚，不只說一半 | 發現說錯時主動更正 | 沒有人看見時，也做真實的選擇 |
| W03 友善 | 主動邀請還沒加入的人 | 用小行動讓別人感到舒服 | 尊重別人不同的玩法與喜好 | 友善也可以保留自己的界線 | 把收到的善意傳給下一個人 |
| W04 合作 | 先找到大家共同的目標 | 依照各自擅長的事分工 | 做不順時互相說明與調整 | 意見不同時一起找第三種方法 | 完成後分享成果與功勞 |
| W05 傾聽 | 停下自己的聲音，讓身體安靜 | 注意表情、動作與沒有說出口的訊息 | 不確定時先詢問，不急著猜答案 | 讓對方慢慢說完，不搶著回應 | 用行動回應真正聽見的需要 |
| W06 嘗試 | 還沒有把握，也願意開始 | 把失敗當成尋找方法的線索 | 一種方法不行，就換一種試試看 | 遇到困難時願意請人幫忙 | 休息、調整，再決定下一步 |
| W07 表達 | 從身體感覺認出自己的心情 | 用清楚的話說出感受 | 不舒服時說「不要、請停下來」 | 具體說出需要什麼幫助 | 選擇適合的時間與方式表達 |
| W08 道歉 | 發現自己的行動影響了別人 | 具體承認做了什麼，不急著找理由 | 聽對方說感受，也尊重對方暫時不接受 | 用實際行動修補造成的影響 | 下次遇到相同情況時做出改變 |
| W09 耐心 | 等待時不急著干擾結果 | 在等待中持續做能做的照顧 | 留心還不明顯的小變化 | 著急時先安頓自己的身體 | 分辨哪些事情能加快，哪些需要時間 |
| W10 選擇 | 先停下來，想清楚真正需要什麼 | 比較不同選擇可能帶來的結果 | 聽聽會受到影響的人的想法 | 做出決定，也接受沒有選到的部分 | 得到新資訊後，可以重新調整選擇 |
| W11 想像 | 從平常事物看見不同可能 | 勇敢提出「如果……會怎樣？」 | 把兩個原本不同的東西組合起來 | 不只照著別人畫，畫出自己的答案 | 把想像變成可以分享或使用的作品 |
| W12 成長 | 回頭看見自己已經做到的改變 | 在新困難中使用以前學過的方法 | 以前需要幫忙的事，現在能自己完成 | 陪伴別人成長，但不替對方完成 | 接受自己仍在學習，帶著勇氣繼續前進 |

### 寫稿前企劃欄位

每篇開寫前先確認：`facet`、`shape`（故事形狀）、`visible_action`、`body_signal`、`boundary_or_safety`、`dominant_sense`、`change_action`、`prologue`（點題小道理）、`voices`／`turns`（多聲線分工）。

## 共通規格

- 共用森林世界，但每集獨立可懂；主角依衝突選擇，不固定輪替。
- 維持 `schema_version: 2`、六個 `sections` 與 `wind_down`。
- 核心詞由角色行動呈現；週末問題問角色做了什麼，不考名詞定義。
- 故事文字遵循 `docs/tts-audio-writing-guide.md`：用短句、自然停頓、適量口語助詞、疊字與聲音詞營造真人說故事的呼吸感。
- 老山羊長故事嵌入既有段落，不新增第七段；成語僅為可選延伸。
